# AI决策

AI Decisioning是Adobe Journey Optimizer B2B Prime背后的智能层。 AI决策不会将每个分支预先构建为静态规则，而是会持续评估用户档案的上下文（包括历程成员资格、参与历史记录、意图得分和实时行为），以确定适合该人员的下一个最佳操作或路径。

这一持续评估将编排从静态规则转向结果驱动的自动化：您无需预先定义每个条件，而是描述您需要的结果，系统确定如何为每个人实现该目标。

## 功能 {#capabilities}

AI决策由以下功能组成：

| 功能 | 它的决定 |
|---|---|
| [历程流量控制](../marketing/journey-traffic-control.md) | 一个人现在应该走哪条路。 当人员符合多个历程的资格时，历程流量控制会在他们的配置文件或行为发生更改时重新路由他们，而不是将他们留在不再适合的路径上。 |
| [下一个最佳路径](../marketing/next-best-path.md) | 历程中人员的最佳路径。 您无需对筛选条件进行硬编码，而是使用自然语言描述意图，系统会在正确的时间将每个人路由到最佳匹配路径。 |
| [发送时间优化](../marketing/email-send-time-optimization.md) | 每个收件人的最佳发送时间范围（基于历史参与度），而不是为每个人制定一个固定计划。 |
| 上下文内容 | 根据内容摘要自动生成的个性化电子邮件变体，用作历程中的单个个性化电子邮件资产。 _即将推出。_ |
| [Brand Concierge](https://experienceleague.adobe.com/zh-hans/docs/brand-concierge/content/home){target="_blank"} | 实时对话路由和响应，如聊天和实时助手。 Brand Concierge需要其他产品权限。 |

其中一些功能可在同一历程中解决不同的问题。 当多个旅程同时竞相吸引同一个人时，历程流量控制可决定哪个旅程优先。 其他功能可决定在人员已进入历程后发生的情况。

## 拆分路径和下一个最佳路径 {#split-paths-next-best-path}

[拆分路径](../marketing/split-merge-paths-nodes.md)允许您使用明确的筛选条件定义历程分支：如果分数高于阈值，则将人员沿一个路径发送，否则沿另一个路径发送。 这种基于规则的逻辑是精确且可完全审核的，但每个新条件都需要一个新分支。

[下一个最佳路径](../marketing/next-best-path.md)节点将AI决策应用于同一个问题。 该模型将共同评估参与度、角色、产品兴趣和funnel阶段，预测每个人的最佳结果，并自动选择最佳路径，而不是固定的阈值。

## 数据输入 {#data-inputs}

AI决策利用以下数据类别：

| 类别 | 示例 |
|---|---|
| 人口统计和第一统计 | 职务、行业和公司规模 |
| 心理学 | 潜在客户得分、紧迫性和优先级 |
| 参与度 | 分数、级别、趋势、上一个活动和渠道 |
| 意图 | 产品或关键词意图，分组到高、中或低存储段 |
| 用户画像 | 交易、职务和角色中的角色 |

## 评估节奏 {#evaluation-cadence}

AI决策使用两个不同的评估计划。 人员的基础配置文件在夜间批处理中重新计算。 决策本身会使用最新夜间配置文件指示的任何信息，实时应用于每次交互。 因此，人工智能决策的连续部分描述了决策时刻，而不是用户档案的刷新率。

## 护栏和规则 {#guardrails-rules}

规则只涵盖某些人明确写下的条件。 当现实落在该树之外时，例如混合角色或无人预料的信号组合，规则将无动于衷，直到有人添加新的分支。 人工智能决策不断对每个人打分，并输出基于置信度的最佳操作，因此它处理组合时没有人为明确编程，并且随着它看到更多结果而改进，而规则永远不会这样做。

规则可以解释并立即用于审核，而AI决策则以置信度得分而不是确定性。 因此，在下列情况下，规则仍然是更好的工具：

* 绝对不能跨越硬合规性边界，例如禁止选择退出的接触。
* 目前还没有足够的容量或历史可供模型学习。
* 每个决定都必须根据要求进行充分解释。

大多数成熟的设置同时运行：规则作为护栏，而AI决策处理其中的所有内容。

## 优点 {#benefits}

* 更少的手动历程复杂性，需要维护的规则分支更少。
* 无需构建数百条规则即可获得更多相关体验。
* 提高资格鉴定效率。
* 更快地确定每个用户档案的下一次最佳参与。

**未来范围：帐户和购买团体上下文**

目前在Journey Optimizer B2B Prime中不可用。 AI决策计划从单个配置文件扩展到：

* 帐户上下文 — 公司和帐户级别的信号作为决策输入。
* 购买团体信号 — 交易中的角色、决策者或从业者状态以及参与购买决策的所有人的参与度汇总。
* 帐户级别的历程编排 — 将帐户或购买组作为一个单元来做出路由和内容决策，并在涉及多个人员时协调历程流量控制。
