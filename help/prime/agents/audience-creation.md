# 为项目创建受众

在[!DNL Adobe Journey Optimizer B2B Prime]中，[_人员列表_](../audiences/people-lists.md)&#x200B;将人员历程的受众定义为自动更新的基于动态筛选器的列表，或具有固定成员资格的静态列表。 在[聊天界面](./chat-interface.md)中，_受众创建_&#x200B;技能通过引导式对话来构建、调整和编辑人员列表。

* **技能** - `audience-creation`和`people-list-comparison`
* **调用** — 直接描述受众条件，上传[!DNL Marketo Engage]智能列表，或命名要编辑的现有列表
* **从**&#x200B;读取/写入 — [!DNL Journey Optimizer B2B Prime]；在调整智能列表时读取[!DNL Marketo Engage]

## 支持的工作流 {#workflows}

该助理支持三个工作流，并可确定哪个工作流适用于您的请求。 如果您的意图不明确，则会在继续操作之前询问。

| 工作流 | 何时使用 | 示例提示 |
|---|---|---|
| **从头开始创建** | 您需要按标准或成员资格定义的新人员列表。 | _“创建北美SaaS公司的营销副总裁（动态）列表。”_ |
| **调整[!DNL Marketo Engage]智能列表** | 您有一个现有的[!DNL Marketo Engage]智能列表或智能营销活动，并且想要一个等效的人员列表。 | _“将此Marketo智能列表调整为人员列表。”_ （附加资产） |
| **编辑现有列表** | 要在已拥有的列表中添加或替换规则。 | _“将规则添加到我的‘企业试用’列表中，以获取50以上的潜在客户得分。”_ |

## 从头开始创建人员列表 {#create-from-scratch}

在生成任何内容之前，助理将确认以下四项。 它要求任何缺失的元素 — 只在一条消息中。

1. **规则/标准** — 列表成员的纯语言描述。
1. **名称** — 该怎样称呼该列表。
1. **位置** — 列表应位于哪个程序中。 提供项目名称，助理会找到该名称；如果有多个匹配项，将要求您进行选择。
1. **类型** — 动态（基于筛选器，自动更新）或静态（固定成员资格）。 这是必需的 — 助理不会猜测；如果未指定，它会询问。

### 动态列表

对于动态列表，助理会主动建议包含个性化属性，以使定位更丰富。 这些属性默认包括&#x200B;**_— 您选择退出，而不是加入_**：

| 属性 | 为什么它有帮助 |
|---|---|
| **派生角色** | 基于角色的内容定位的人工推断购买者角色。 |
| **派生意图** | 推断的购买意图信号出现在市场账户中。 |
| **参与层** | 计算出的参与度级别，可确定参与联系人的优先级。 |

在继续操作之前，请告知助理是否要删除其中的任何内容。

### 静态列表

* **静态，无条件** — 列表是空的，可供您手动添加成员。
* **静态来自条件（快照）** — AI Assistant构建匹配集并复制其中的人员。 填充是异步的 — 助理会确认列表已创建，但请注意，人员可能需要几分钟才能显示。 它不会立即声明该列表已准备就绪。

## 审核卡片 {#review-card}

在您批准之前，不会创建任何内容。 在说明您的条件后，助理会显示一个交互式&#x200B;_人员列表创建审核_&#x200B;信息卡（对于[!DNL Marketo Engage]列表中的改写，该信息卡标题为&#x200B;_人员列表转化审核_）。

信息卡中的每一行表示一个条件：

| 列 | 含义 |
|---|---|
| **要求**（或自适应的[!DNL Marketo Engage]列表名称） | 您的原始请求或源Marketo过滤器。 |
| **（规则）** | 为该条件生成的基于属性的实际规则。 |
| **包含** | 用于保留或删除该规则的复选框。 |

**置信度级别：**

* **高置信度**&#x200B;行完全匹配且默认选中。
* **低置信度**&#x200B;行（近似映射或标记的任何内容）显示有通知指示符，默认情况下未选中。
* 系统无法映射的行显示&#x200B;**“未找到等效项”** — 这些行没有规则且保持未选中状态。

_转化摘要_&#x200B;与&#x200B;_N高置信度_&#x200B;和&#x200B;_N低置信度_&#x200B;匹配，并提示：默认情况下未选中低置信度规则；选中它们以包含或描述您希望在聊天中进行的更改。

**卡片操作：**

* **继续** — 仅使用选中的规则创建列表。
* **在聊天中描述您想要的更改** — 使用&#x200B;_预填充输入我想更改：“_”以便您能够进行优化；AI助手重新生成并显示一张新卡片，保留您已批准的规则。

您还可以随时键入跟进（例如，_“还限制为员工数超过500人的公司”_），AI助手将重新生成卡。

## 属性映射 {#attribute-mapping}

当您描述标准时，助理会将每个条件转换为实际的、已知的人员级别属性。 Review卡上可能会显示三个结果：

1. **匹配（高置信度）** — 您的条件直接映射到属性（例如，_“email is acme.com”_&#x200B;映射到`email`属性）。 默认选中。
1. **近似（低置信度）** — 在名称或数据模型中可用的最接近的属性不同（例如，Marketo _金额_&#x200B;筛选器近似为&#x200B;_潜在客户得分_）。 显示时带有说明差异的注释；默认情况下未选中。
1. **未找到** — 条件无法映射到任何已知属性。 显示为&#x200B;_“未找到等效项”_；未生成规则。

这就是为什么您描述的列表可能会返回比您指定的条件少的规则 — 不匹配的条件会显式出现，而不是静默地丢弃。 如果重要标准显示为“未找到”，请使用属性的真实名称重新短语，然后助手会重试。

>[!NOTE]
>
>如果要将电子表格列映射到字段（具有&#x200B;_Source列_、_目标字段_、置信度百分比和&#x200B;_未映射列_&#x200B;列表的字段映射卡片），即潜在客户导入流程，而不是受众创建。 查看[导入潜在客户技能](./skills.md#audiences-people)。

## 编辑现有列表的规则 {#edit-rules}

当要求更改已拥有的列表中的规则时，助理将建立哪个列表和哪个编辑模式：

* **添加/附加** （默认为&#x200B;_“添加规则”_、_“添加更多规则”_） — 新规则与现有规则合并。
* **替换** （默认值&#x200B;_“替换规则”_，_“将规则更改为”_） — 新规则替换列表中的所有现有规则。

该助理汇总将应用的内容，并清楚地声明它是添加还是替换，然后要求您在提交之前确认。 应用后，它会报告规则总数以及添加或替换的规则数。

>[!NOTE]
>
>编辑内容使用合并感知路径，因此“添加”操作绝不会自动覆盖现有规则。

## 受众重叠 {#overlap}

询问AI助手比较两个人员列表（例如，_“向我显示‘第3季度网络研讨会’和‘企业帐户’之间的重叠”_），它将呈现&#x200B;_人员列表重叠_&#x200B;卡：

* 显示计数&#x200B;**“{N}”的标题徽章（共同）。“**”
* 每个列表成员总数和重叠为A的&#x200B;**&quot;X% · B的Y%&quot;**&#x200B;的统计信息行。
* 两个列表中人员的成员表，具有&#x200B;**Name**&#x200B;列和可以控制的第二列 — **电子邮件**（默认）、**公司**&#x200B;或&#x200B;**职务**，具体取决于您询问的内容。
* 单击任意名称以在工作区中打开该人员。
* 如果没有重叠，卡片会明确显示： _“这两个列表之间没有共同的成员。”_

**限制：**

| 限制 | 详细信息 |
|---|---|
| **表大小** | 显示最多200个成员；超出此限制，它将记下&#x200B;_“显示200个N — 要求我优化查询以缩小结果范围。”_ |
| **重叠计算** | 根据电子邮件地址进行计算；没有电子邮件的用户将从交叉点中排除。 |
| **列表大小** | 读取每个列表最多大约1,000个成员。 对于较大的列表，助理会告诉您结果不完整。 |
| **草稿动态列表** | 无法比较 — 尚未发布的列表没有实时区段。 助手会要求您先发布或使用静态列表。 |

## QA验证 {#qa-validation}

创建或更新列表后，AI Assistant提供： _“是否要我验证该列表是否已正确配置？”_ 如果接受，它会重新获取列表并报告以下检查：

| Check | 结果 |
|---|---|
| 在正确的程序/文件夹下找到的列表 | 通过/失败 |
| 过滤器计数与应用的过滤器计数匹配 | _N_&#x200B;个筛选器/不匹配 |
| Personalization属性存在（如果包含） | 存在/缺失 |
| 列表名称与您请求的名称匹配 | 通过/失败 |
| 估计成员数 | _count_&#x200B;或不适用 |

## 限制 {#limitations}

| 限制 | 详细信息 |
|---|---|
| **来自[!DNL Marketo Engage]**&#x200B;的静态列表适配 | 您无法将[!DNL Marketo Engage]静态列表（或电子邮件或其他非筛选资源）调整为人员列表。 静态列表是明确的成员ID，不能以过滤器形式表示；助理会请求智能列表或智能营销活动。 |
| **基于活动和成员资格的筛选器** | 当从[!DNL Marketo Engage]进行调整时，诸如&#x200B;_电子邮件之类的筛选器已打开_、_已访问网页_、_表单已填写_、_列表成员_&#x200B;以及&#x200B;_智能营销活动成员_&#x200B;没有对等人员列表并返回为“未找到对等人员”。 |
| **公司级条件** | 尽可能转换为最接近的人员级别属性（人员列表对人员属性进行操作），并在配合松动时标记为低置信度。 |
| **深度嵌套的AND/OR逻辑** | 复杂的嵌套逻辑可能会折叠为顶级AND/OR；发生这种情况时，助理会记录此信息。 |
| **名称冲突** | 未自动解析 — 如果获取了名称，则助手会要求您输入其他名称，而不是静默附加后缀。 |
| **需要批准** | 在您单击&#x200B;**[!UICONTROL 继续]**、确认或给出明确的指示之前，助理将不会创建或修改列表（_&quot;已批准&quot;_，_&quot;看起来不错&quot;_，_&quot;构建它&quot;_）。 |
| **静态快照填充** | 根据标准创建的静态列表中的成员资格会在几分钟内填充 — 不会立即填充。 |
