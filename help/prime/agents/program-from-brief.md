# 从简介创建项目

在[!DNL Adobe Journey Optimizer B2B Prime]中，[_项目_](../marketing/programs.md)&#x200B;是营销活动中历程、人员列表、令牌和配置的顶级容器。 从[聊天界面](./chat-interface.md)，计划创建技能在一个引导式对话中从您的简报（计划本身、子文件夹、令牌、人员列表和人员历程）端到端地构建整个结构。

* **技能** - `program-creation`
* **调用** — 上传一份简报并输入&#x200B;_根据这份简报创建项目_ / _根据简报生成项目_，或直接描述该营销活动
* **从**&#x200B;读取/写入 — [!DNL Journey Optimizer B2B Prime]；还从租户读取程序类型配置

## 简要上传

单击聊天输入中的&#x200B;_附加_&#x200B;图标以上传文件，然后描述您想要的内容。 系统会提取简报的文本，并将其作为上下文传递给助手(您会在聊天中看到“营销活动简报已上传：文件名(NkB)”指示器)。

### 支持的文件类型

| 格式化 | 行为 |
|---|---|
| `.txt`, `.md` | 直接在浏览器中读取 |
| `.pdf`, `.docx`, `.xlsx`, `.json`, `.csv` | 已发送到后端文本提取服务 |

名称包含&#x200B;*brief*&#x200B;的文件会自动标记为营销活动摘要；否则，助理会从上下文推断该文件。

### 大小限制

在到达助理之前，简短文本将被截断为约30,000个字符（约7.5K个令牌）。 超长简报在该点处被切断（由`(truncated to 29KB)`便笺指示）。 预先提供重要的营销活动详细信息。

## 推荐的简短内容

流量引入读取以下输入的摘要 — 尽可能包括：

* 项目名称和简短描述/目的
* 计划类型信号（如商展、活动、网络研讨会、路演、会议或培养）
* 人员列表的受众标准
* 历程设计 — 计数、进入标准、接触点/计时和上线日期
* 令牌（可重复使用的值，例如活动日期、地点、主题行）

AI助手会在构建之前请求任何缺失的内容。

## 构建阶段

该技能分三个阶段运行：**INTRATION→APPROVE → BUILD**。 在您批准之前，不会创建任何内容。

### 摄取

助理从简介中提取以下内容，并提示缺少任何内容：

* 项目名称
* 父文件夹（如果未指定，则默认为工作区根）
* 描述
* 项目类型（从可能的简短措辞中推断）
* 要创建的令牌（名称、类型、值）
* 人员列表受众标准
* 历程 — 计数、进入条件、接触点、上线日期

### 批准

助理在继续操作之前会显示确认摘要：

`I'll create {program} in folder {id}, with {N} token(s), {N} dynamic people list(s), and {N} journey(s). Shall I proceed?`

在您通过明确的继续操作（如&#x200B;_继续_、_继续_、_生成工具_、_已批准_&#x200B;和&#x200B;_是_）做出响应之前，不会调用任何创建工具。

### 生成

构建步骤按固定顺序运行；每个步骤都取决于上一步生成的ID。

| 步骤 | 描述 | 工具 | 输出 |
|---|---|---|---|
| **1. 文件夹** | 按名称解析父文件夹或使用工作区根；可以创建新的子文件夹 | `getRootTree`, `browseFolders`, `createFolder` | 文件夹名称+ ID |
| **1.5. 项目类型** | 查找租户项目类型并挑选匹配项 | `browseProgramTypes` | 键入名称+ `programTypeId` |
| **2. 项目** | 在已解析的文件夹下创建程序，绑定到所选类型 | `createProgram` | 项目名称、ID、类型 |
| **3. 令牌** | 在程序中创建每个`my.*`令牌 | `createProgramToken` | 令牌名称 |
| **4. 人员列表** | 根据受众标准生成基于属性的规则并创建动态人员列表 | `generate_ruleset`, `createSmartListWithRules` | 列表名称+ `smartListId` |
| **5. 历程** | 通过连接到步骤4列表的人员受众登入节点创建每个人员历程 | `create_journey` | 历程名称、ID |
| **6. 发布** | 设置并确认上线日期后，发布或计划历程 | `publish_journey_with_dates` | 实时/计划历程 |
| **7. QA** | 可选的验证预览和完整QA检查 | `qa_program_preview`, `qa_program` | 通过/失败/警告报告 |

**_步骤依赖项:_**

* 必须先提供程序编号（步骤2），然后才能附加令牌、列表或历程。
* 步骤4中的`smartListId`在步骤5中传递到`create_journey` — 省略它会使历程的受众进入节点为空。
* 历程提示始终以&#x200B;_Start with a Person受众节点作为入口点_&#x200B;开始，以确保填充的入口节点。

## 输出资产

### 项目

顶级容器。 其详细信息视图会显示以下选项卡：

| 选项卡 | 内容 |
|---|---|
| **概述** | 名称、描述、程序类型、状态、文件夹位置、时间戳 |
| **属性** | 类型定义的自定义字段（仅当程序类型启用时） |
| **令牌** | 此项目范围的`my.*`令牌 |
| **历程** | 项目群中包含的历程 |

### 令牌

范围限定为程序的可重用`my.*`值（例如`my.eventDate`）。 每个类型都有一个类型 — `text`、`date`、`rich text`、`score`或`number`之一 — 和一个值。 它们解析项目范围内的内部电子邮件和内容。

### 人员列表

动态（智能）人员列表，通过项目下创建的基于属性的生成规则，根据受众标准生成。

### 历程

人员历程从绑定到步骤4人员列表的人员受众进入节点开始，然后是简介中的接触点（例如，_在1天后发送欢迎电子邮件，在3天后跟进_）。 如果设置了上线日期，则可以立即发布旅程或安排旅程。

## 项目类型解析

程序在创建时将&#x200B;**永久绑定**&#x200B;到租户定义的程序类型 — 此后无法更改。 在每种情况下，流都通过`browseProgramTypes`显式解析类型。

| Case | 行为 |
|---|---|
| **多个可用类型** | 将简短措辞与类型匹配（例如，贸易展/展位/展览= *贸易展*/*活动*；网络研讨会= *网络研讨会*/*活动*；nurture/drip = *Nurture*；无清除信号= *默认*）。 如果未找到匹配项，助理将列出可用类型并询问。 |
| **仅默认租户** | 使用&#x200B;*Default*，并注意管理员可以添加自定义[程序类型](../admin/program-types.md)。 |
| **未配置任何类型** | 停止 — 创建将失败。 在重试之前提示管理员配置程序类型。 |

## 默认值

| 设置 | 默认 |
|---|---|
| **文件夹** | Workspace根目录（如果未指定文件夹） |
| **项目类型** | *默认值*（如果没有简短的信号和手动选择） |
| **Personalization属性** | 默认情况下包含派生角色、派生意图、参与层（提供选择退出功能） |
| **发布** | 对于未来的上线日期不是自动的 — 标记为手动`ACTIVATE journey by {date}`步骤；仅在现在启动并确认后立即发布 |
| **审批门** | 在明确批准之前不会创建任何资源 |

## 限制

| 限制 | 详细信息 |
|---|---|
| **简短长度上限** | ~30,000个字符；截断的摘要将丢失尾随内容 — 将基本内容放在最前 |
| **不可变程序类型** | 创建后无法更改；请在批准前验证正确的类型 |
| **必需的内部字段** | `parent`和`programTypeId`始终已设置；省略`parent`会导致访问被拒绝；省略类型静默回退到&#x200B;*默认值* |
| **历程需要人员列表** | 步骤4中的`smartListId`对于步骤5是必需的；流强制实施此操作 |
| **异步列表成员资格** | 静态（来自标准）列表会在几分钟内填充 — 不是立即填充 |
| **错误处理** | 报告工具失败时带有确切的错误；助理将提示确认输入并重试 |
| **名称冲突** | 未自动解析 — 将请求其他名称 |
| **作用域** | 仅限Journey Optimizer B2B Prime；Marketo用户应使用单独的程序创建/程序规划技能 |


## QA检查

构建完成后，AI Assistant提供：

> “您是否希望我在启动之前运行QA检查以验证所有内容？”

确认运行`qa_program_preview`，然后运行`qa_program`。 该进程会返回检查通过、失败和任何警告的报告，以及建议的后续步骤。
