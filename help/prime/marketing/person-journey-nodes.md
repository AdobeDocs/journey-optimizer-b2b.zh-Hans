# 人员历程节点

创建人员历程后，添加受众并使用节点构建历程。 历程画布提供了一个可视设计空间，您可以在其中构建多步骤B2B营销用例。

_[!UICONTROL 人员受众]_&#x200B;节点自动成为历程中的第一个节点。 选择受众后，通过将不同的操作、事件和决策节点组合为多步骤、跨渠道方案来构建旅程。 历程的每个节点表示逻辑路径上的一个步骤。

:::: landing-cards-container

:::
![人员受众节点](https://cdn.experienceleague.adobe.com/icons/users.svg)

人员受众节点

历程始终从人员受众节点开始。 使用静态或动态人员列表设置输入受众，以定义哪些人员配置文件进入历程。

[了解人员受众节点](./person-audience-node.md)
:::

:::
![操作节点](https://cdn.experienceleague.adobe.com/icons/circle-play.svg)

操作节点

将更改应用于路径上的每个人，例如发送电子邮件或更新数据值属性。

[了解操作节点](./action-nodes.md)
:::

:::
![事件节点](https://cdn.experienceleague.adobe.com/icons/puzzle-piece.svg)

事件节点

发生特定活动或数据更改（如打开电子邮件或数据值更改）时，将人员前进到下一个历程步骤。

[了解事件节点](./listen-for-event-nodes.md)
:::

:::
![等待节点](https://cdn.experienceleague.adobe.com/icons/clock.svg)

等待节点

在将人员转至下一步之前，在设置的持续时间或特定日期和时间内暂停历程进度。

[了解等待节点](./wait-nodes.md)
:::

:::
![拆分和合并路径节点](https://cdn.experienceleague.adobe.com/icons/code-branch.svg)

拆分和合并路径节点

根据条件将人员划分为单独的路径，对每个路径应用不同的操作或事件，然后合并路径以继续旅程。

[了解拆分和合并路径节点](./split-merge-paths-nodes.md)
:::

:::
![下一个最佳路径节点](https://cdn.experienceleague.adobe.com/icons/lightbulb.svg)

下一个最佳路径节点

使用AI驱动型决策将每个人路由到最相关的路径。 使用自然语言描述您的意图，让系统评估用户档案上下文、参与历史记录和帐户数据来做出路由选择。

[了解下一个最佳路径节点](./next-best-path.md)
:::
